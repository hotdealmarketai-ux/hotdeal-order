# 핫딜마켓 발주 플랫폼

핫딜마켓 가맹점과 서부일광 소매업자가 발주를 넣고, 4개 업자(서부일광·장흥·채움채·새롭)가 발주를 받는 모바일 우선 웹앱.
발주를 넣으면 AI(또는 규칙기반)가 품목·수량·부연설명을 정갈하게 정리해 영수증 형태의 발주서를 만든다.

> 기존 `hotdeal-market` 본체 / `saerop-b2b` 와 **완전히 분리된 신규 프로젝트**입니다.

## 기술 스택
- Next.js 16 (App Router, Turbopack) · React 19 · TypeScript
- Auth.js v5 (아이디/비밀번호, JWT 세션 30일 자동로그인)
- Prisma 6 + PostgreSQL (로컬: 직접 띄운 PG / 운영: Supabase)
- AI 정리: Anthropic Claude (키 없으면 규칙기반 자동 폴백)
- 디자인: 현대카드st 흑백 모노톤, 어르신 가독성(큰 글씨·큰 버튼)

## 포트 (기존 작업과 겹치지 않게)
- 앱: **4300** (`npm run dev`)
- 로컬 Postgres: **5433**

## 로컬 실행
```bash
# 1) 로컬 Postgres 기동 (이미 .localpg 가 구성돼 있음)
npm run db:start
# 2) 스키마 반영 + 시드(테스트 계정/샘플)
npm run db:migrate   # 최초 1회 (이미 적용됨)
npm run db:seed
# 3) 개발 서버
npm run dev          # http://localhost:4300
```

## 발주 라우팅
| 카테고리 | → 받는 업자 |
|---|---|
| 과일 | 서부일광 |
| 야채 | 장흥 |
| 공구(공동구매) | 새롭 |
| 두부·콩나물·순두부·숙주 | 채움채 |

- 핫딜마켓 가맹점 = 4개 카테고리 전부 / 서부일광 소매 = 과일만(+픽업시간)
- 새롭(관리자) = 전체 발주 열람 + 핫딜마켓 전용 필터 + 가입승인 + 재고현황 작성
- 핫딜마켓 가맹점만 = 하단 네비 '재고현황' 열람(새롭이 작성)

## AI 발주 정리
- `ANTHROPIC_API_KEY` 가 있으면 Claude가 품목/수량/부연설명을 사무적으로 정리.
- 없으면 규칙기반(공백·중복 정리)으로 자동 폴백 → **키가 없어도 앱은 그대로 동작**.
- 원칙: 의미(품목·수량)는 바꾸지 않고 표기만 다듬음. 원문은 항상 보존(영수증 '원본 보기').
