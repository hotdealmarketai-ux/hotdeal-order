# 오더야 (Orderya) — 플랫폼 개발 · 릴리즈 보고서

> B2B 청과·농산물 공동구매 발주 웹 플랫폼
> 라이브: https://www.orderya.xyz (hotdeal-order-jet.vercel.app)

---

## 1. 서비스 개요

오더야는 **핫딜마켓 가맹점**과 **서부일광 소매 거래처**가 매일 필요한 농산물을 한 곳에서 발주하면, AI가 발주서를 정갈하게 정리해 각 **공급 업자**에게 자동으로 전달하는 B2B 발주 플랫폼입니다.

**발주 흐름**

```
점주(가맹점·소매)  →  발주 입력(칸 직접입력 / 카톡식 채팅)
        ↓ AI가 품목·수량·비고를 표준 발주서로 정리
        ↓ 카테고리별로 알맞은 업자에게 라우팅
업자(서부일광·장흥·채움채·새롭)  →  발주 확인 → 점주에게 확인 알림(웹푸시)
        ↓
관리자(새롭 본사)  →  전체 발주 관리·취합·회원 승인
        ↓
채움채(두부류)  →  외부 발주 시스템에 자동 제출(스케줄러)
```

**핵심 가치**
- 카톡처럼 자유롭게 적어도 AI가 표준 발주서로 정리
- 매일 마감(오후 8시)·운영시간 자동 관리 + 마감 카운트다운
- 웹푸시로 발주 도착/확인/마감을 실시간 알림 (PWA, 홈 화면 설치)
- 점주에게는 실제 공급 업체명을 숨김(과일/야채/공구/채움채로 표기)

## 2. 기술 스택 & 아키텍처

| 영역 | 사용 기술 |
|---|---|
| 프레임워크 | Next.js 16 (App Router, Turbopack), React 19 |
| 인증 | Auth.js v5 (next-auth) — Credentials + JWT 세션 |
| 데이터베이스 | PostgreSQL + Prisma 6 (운영: Neon, 로컬: PostgreSQL 16) |
| AI | Anthropic Claude (`@anthropic-ai/sdk`) + 규칙기반 폴백 |
| 웹푸시 | Web Push (VAPID), 서비스워커 |
| 예약작업 | GitHub Actions cron → Vercel API 호출 |
| 스타일 | Tailwind CSS 4 + 커스텀 디자인 토큰 |
| 배포 | Vercel + 커스텀 도메인(orderya.xyz, 가비아 DNS) |
| 파일저장 | Supabase Storage(사업자등록증), 없으면 로컬 폴백 |

**관통하는 엔지니어링 원칙**
1. **Graceful degradation** — AI·푸시·외부연동은 키가 없거나 실패해도 규칙기반 폴백/무시로 앱이 항상 동작.
2. **KST 명시 계산** — 서버 타임존(UTC)과 무관하게 한국시간으로 통일.
3. **원본/정리본 동시 보존** — AI 정리가 틀려도 점주 원본을 추적 가능.
4. **업체명 비공개** — 점주 화면에서만 실제 업체명을 숨기되 라우팅은 정상.
5. **Vercel Hobby cron 한계 우회** — GitHub Actions + 보안토큰으로 예약작업.

## 4. 페이지별 기능 명세

### 4.1 진입 · 인증

#### `/` (루트 리다이렉터) — 전체
화면 없이 로그인 여부·역할·승인상태로 분기. 비로그인→`/login`, 미승인→`/pending`, 업자/관리자→`/vendor`, 점주→`/order`.

#### `/login` — 비로그인
- **폼**: 아이디 / 비밀번호 / "자동 로그인" 체크박스(기본 켜짐) → `loginAction`. 실패 시 "아이디 또는 비밀번호가 올바르지 않습니다."
- **버튼**: `로그인`(처리 중 "로그인 중…"), `가입 신청`(→ `/signup`).
- 자동 로그인 체크 시 세션 ~400일 유지, 미체크 시 12시간 후 만료.

#### `/signup` — 비로그인
- **폼**: 상호명* / 연락처* / 업장 소재지* / 사업자등록증(사진촬영·파일, 선택) / 아이디(영숫자 3자+)* / 비밀번호(4자+)* → `signupAction`.
- **버튼**: 뒤로(`‹`), `가입 신청하기`(처리 중 "신청 중…").
- 검증 통과 시 `APPLICANT`/`PENDING`으로 생성 → 자동 로그인 → `/pending`. 사업자등록증 저장은 실패해도 가입 진행.

#### `/pending` — 미승인 로그인 사용자
- 상태별 안내: 승인 대기 / 계정 정지 / 가입 반려. 상호명·아이디·상태 배지 표시.
- **버튼**: `로그아웃`. 승인되면 새로고침 시 발주 화면으로 자동 이동.

### 4.2 점주 화면 (가맹점 · 소매)

> 공통: 하단 네비 `발주 / 재고현황 / 마이`. **재고현황 탭은 핫딜마켓 가맹점만** 노출(서부일광 소매는 발주·마이 2탭).

#### `/order` (메인 발주 작성)
- **마감 카운트다운**(핫딜 가맹점만): "금일 발주 마감은 오후 8시 입니다." + 실시간 남은시간.
- **발주 알림 토글**(`PushToggle`): `알림 켜기` / `끄기` — 웹푸시 구독.
- **발주 입력 — 2가지 방식**
  - **칸 직접입력(grid)**: 카테고리 탭(과일/야채/공구/채움채) 전환, 품목·수량·설명 칸. 줄 추가는 자동, `삭제` 버튼.
  - **채팅 발주(`채팅으로 발주하기`)**: 카톡처럼 자유롭게 적으면 `발주` 버튼으로 AI가 카테고리별로 분해. `칸에 직접 입력하기`로 복귀.
  - **채움채(TOFU)**: 자유입력이 아닌 **고정 5품목 체크리스트**(수량만) — 비타 수입 봉지콩 300g / 아삭채움 숙주봉지 500g / 강릉두부 500g / 강릉순두부 600g 몽글 / 맑은결 손두부 500g.
- **버튼**: `발주하기` → 확인 단계("이대로 발주할까요?") → `네, 발주할게요`(`createOrderAction`).
- **특이 동작**
  - 운영시간 잠금(핫딜만): 평일 12–20시, 토 12시~일 20시 연속, 일 휴무. 닫히면 입력 잠금.
  - 이번 발주창에 이미 발주가 있으면 새 발주를 막고 "발주수정만 가능합니다."
  - 받는 곳 라벨: 핫딜 가맹점은 과일/야채/공구/채움채로 **실제 업체명 비공개**, 서부일광 소매는 실명.

#### `/order/[id]` (단일 발주서)
- 영수증 형태 표시 + `ReceiptCard`의 `원본 보기`/`정리본` 토글(점주 원본 vs AI 정리본).
- **버튼**: `발주 수정`(수정 가능할 때만), `새 발주하기`.
- 과거일이면 "발주가 완료되었습니다." + 수정 차단. 확정 시 "발주가 확인되었습니다. 준비 중이에요."

#### `/order/[id]/edit` (발주 수정)
- 기존 발주 품목/픽업시간 수정 → `수정 완료` → 확인 → `네, 수정할게요`(`updateOrderAction`).
- 수정하면 받는 업체에 '발주수정' 푸시 + **확인 상태 리셋**(재확인 필요). 운영시간/이번 창 발주만 수정 가능.

#### `/order/day/[date]` (일자별 합본 발주서)
- 핫딜 가맹점이 하루에 넣은 4종 발주를 한 화면에 모아 카테고리 순으로 표시.
- **버튼**: 각 카테고리 `수정`(수정 가능한 발주만), `새 발주하기`.
- 과거일이면 "완료" 배지, 오늘 확정 건은 "확인됨 · 준비 중".

#### `/mypage` (마이페이지)
- 프로필 카드(상호·역할·아이디·연락처·소재지) + 지난 발주 내역.
- 핫딜은 날짜별로 묶어서, 소매는 개별 발주로 표시. 과거일은 "완료" 배지.
- **버튼**: `프로필 수정`, 각 발주 행, `로그아웃`.

#### `/mypage/edit` (프로필 수정)
- **폼**: 상호명 / 연락처 / 소재지 / 새 비밀번호(변경 시에만) → `updateProfileAction`. 비번은 입력 시 4자 이상.

#### `/inventory` (재고현황 — 핫딜 가맹점 전용)
- 본사가 등록한 재고를 읽기 전용으로 열람. 상태에 따라 배지 색(넉넉/부족/품절).
- 안내: "재고는 실시간 반영이 되지 않으므로, 실제와 다를 수 있습니다."

### 4.3 업자 화면 (서부일광 · 장흥 · 채움채 · 새롭)

#### `/vendor` (들어온 발주 목록)
- 본인에게 라우팅된 발주를 날짜별로 표시. 점주 상호·시각·카테고리·품목수·픽업시간.
- **버튼**: 날짜바 `‹`/`›`/날짜선택/`오늘`(미래 선택 불가), `오늘 전체주문 집계 보기`(→집계), 각 발주 행(→상세), `알림 토글`, `로그아웃`(새롭 제외).
- 상태 배지: 점주 수정·미확인이면 빨강 `발주수정`, 확인됨이면 `발주 확인`.

#### `/vendor/[id]` (발주서 상세 + 확인)
- `ReceiptCard`로 발주 내역 + 정리본/원문.
- **버튼**: `발주 확인`(→점주에게 확인 알림) / `확인 취소`. 점주가 수정 후 미확인이면 "내용 확인 후 다시 발주 확인을 눌러주세요." 안내.

#### `/vendor/summary` (집계 발주서)
- 여러 점포 발주를 **[품목 > 종류] 2단계**로 묶어 총량 표시(사입·입찰용).
- 과일/야채는 품종까지 정확히 분류(부사·시나노골드=사과 / 샤인머스캣·어텀크리스피=포도), 공구·두부는 같으면 묶기.
- **버튼**: `발주 레포트 생성`(서부일광만 → 입찰 준비 리포트).

#### `/vendor/report` (경매 입찰 준비 리포트 — 서부일광 전용)
- 그날 발주를 토대로 "[항목] — [수량] 입찰 준비" 문장 자동 생성. 등급(다이·특 등)은 괄호로.

### 4.4 관리자 화면 (새롭 본사 · `ADMIN_SAEROP` 전용)

> 하단 네비: `공동구매 발주`(/vendor) ↔ `관리`(/admin).

#### `/admin` (관리자 홈)
- 메뉴: `가입 대기`(대기 건수 배지) / `회원 관리` / `전체 발주 목록`(오늘 N건) / `핫딜마켓 발주관리`(오늘 N건) / `재고`.
- **N건 = 오늘 주문한 점포 수**(중복 제거) — 한 점포가 여러 종을 넣어도 1.

#### `/admin/approvals` (가입 승인)
- 대기 신청자 카드별로: `핫딜마켓 가맹점` / `서부일광 소매`(승인+역할 배정) / `반려`. 사업자등록증 `보기` 링크.

#### `/admin/members` · `/admin/members/[id]` (회원 관리)
- 목록: 전체 회원을 상태·역할 배지와 함께. 상세에서:
  - **폼**: 상호/연락처/소재지/역할/상태 수정(`저장하기`), 비밀번호 초기화(`비밀번호 변경`).
  - **버튼**: `계정 정지`/`정지 해제`, `회원 삭제`(2단계 확인 → 발주 이력까지 삭제).
  - **본인 계정 보호**: 본인이면 역할·상태 잠금, 삭제·정지 숨김(자기잠금 방지).

#### `/admin/orders` (전체 발주 목록) · `/admin/hotdeal` (핫딜마켓 발주관리)
- 범위 탭(전체/업자별 또는 카테고리별) + 날짜바로 조회.
- **버튼**: `발주 취합 보기`(→집계), 합본 행(→`/admin/combined`), 개별 행(→`/admin/orders/[id]`).
- 핫딜 가맹점 발주는 점포×날짜로 4종을 합본 처리.

#### `/admin/orders/[id]` · `/admin/combined/[userId]/[date]`
- 개별 발주서 상세 / 한 점포의 하루 4종 합본 발주서(인쇄 가능, `PrintButton`).

#### `/admin/summary` (취합/집계)
- 선택 컨텍스트·스코프·날짜의 발주를 AI로 품목·종류별 묶어 거래처 발주서 형태로 표시.

#### `/admin/inventory` (재고현황 작성)
- **폼**: 품목 `추가하기`, 항목별 상태/수량 `저장`, `삭제`. 변경 시 가맹점 열람 화면도 즉시 반영.

### 4.5 API · 예약작업

#### `/api/auth/[...nextauth]` — NextAuth 인증 핸들러
Credentials 로그인, JWT 세션. **역할/상태는 매 요청 DB에서 최신값**을 읽어 승인·역할 변경이 즉시 반영.

#### `/api/cron/notify` — 발주 알림 (CRON_SECRET 보호)
핫딜마켓 가맹점에게 마감 1시간 전(19시) / 마감(20시) / 발주 시작(12시) 푸시.

#### `/api/cron/submit-chaeumchae` — 채움채 외부 자동발주 (CRON_SECRET 보호)
그 발주창의 두부류 주문을 합쳐 외부 채움채 시스템(hancuc.co.kr)에 자동 제출. 실패 시 관리자에게 알림.

**GitHub Actions 스케줄(KST)**: 19:00 마감경고 · 20:00 마감 · 12:00 발주시작 · 20:05 채움채 자동제출.

## 5. 핵심 구현 내역 (어떻게 구현했나)

### 5.1 AI 발주 정리 (의미 불변, 표기만 정리)
- `normalizeOrder` — 품목명 오타·줄임말 교정("샤인"→"샤인머스캣"), **과교정 절대 금지**(불확실하면 원문 유지). AI 항목 수가 입력과 다르면 규칙기반으로 폴백(인덱스 어긋남 방지).
- `parseChatOrder` — 카톡식 자유문장을 품목으로 분해 + 카테고리 자동 분류. 두부류는 채팅에선 야채로(채움채는 체크리스트 전용).
- `aggregateOrders` — 여러 지점 발주를 [품목>종류] 2단계로 묶기.
- `auctionReport` — 경매 입찰 준비 문장.
- **항상 폴백**: Claude 키 없거나 실패 시 규칙기반으로 자동 전환 → 앱은 늘 동작.

### 5.2 수량 처리 규칙
- **숫자/박스/개 → 숫자 하나**로 정리("3박스"→"3"), **kg·알만 단위 유지**("10kg", "5알"). 영수증 표시는 `displayQty`가 결정적으로 적용.
- **다이(N다이)는 '설명'이지 수량이 아님** — "한 박스에 든 낱개 개수(크기·등급)"를 뜻하므로 **비고(note)로 분류**하고, 실제 수량은 박스 수. 예) "부사 3다이 전반 4박스" → 품목 *부사 사과* / 수량 *4* / 비고 *3다이 전반*.
- **원본/정리본 동시 보존** — 점주가 입력한 원문(rawQty "3박스")과 정리본(qty "3")을 모두 저장, 영수증 `원본 보기`로 확인.

### 5.3 시간 · 운영시간
- 모든 날짜/시각은 KST 기준(`kstToday`, `kstDateOf`, `shiftDate` 등) — 서버 UTC와 무관.
- 운영시간: 평일 12–20시, 토 12시~일 20시 연속, 일 휴무. 핫딜 가맹점만 적용, 소매는 상시.
- 마감 카운트다운은 매초 갱신, 마감 후엔 다음 발주 시작까지 표시.

### 5.4 웹푸시 (PWA)
- VAPID 기반, 서비스워커는 **푸시 전용(캐싱 안 함)**으로 데이터 항상 최신.
- 새 발주 도착(업자), 발주 확인(점주), 주문 완료(점주), 마감/시작(가맹점) 알림. 만료된 구독은 자동 정리.

### 5.5 채움채 외부 자동발주
- 외부 Laravel 시스템(hancuc.co.kr) 세션을 키로 확보 → CSRF 토큰 추출 → 품목별 수량을 자동 제출. 마감 5분 뒤(20:05) 스케줄러가 실행, 실패 시 관리자 푸시.

### 5.6 보안 · 안정성
- 비밀번호 bcrypt 해시, 모든 서버 액션에 역할 가드(`requireMerchant/Vendor/Admin`).
- 사업자등록증 업로드는 확장자 화이트리스트 외엔 `.bin` 저장(저장형 XSS 차단), 10MB 제한.
- DB 장애 시에도 앱이 죽지 않고 로그인 화면으로(`getCurrentUser` null 처리).
- 본인 계정 자기잠금 방지(관리자가 자신을 강등·정지·삭제 불가).

## 6. 릴리즈 타임라인 (개발 내역)

> 시간순 주요 마일스톤 (커밋 기준 최신 → 과거)

**입력·정리 고도화**
- 다이 학습: 다이는 수량이 아니라 비고(크기·등급 설명)로, 실제 수량은 박스
- 영수증 수량 표시 규칙: 숫자/박스/개→숫자, kg·알만 단위 유지
- 박스 자동변환 제거(숫자만/숫자+단위 구별만)
- 발주 화면 다듬기 16건 (채팅 문구·재고 여백·날짜바·받는곳 역할별·채움채 라벨)
- 홈 개선 8건 (채움채 체크리스트·회원삭제·점포수 집계·자동로그인·받는곳 비공개)

**채움채 · 알림 · 운영**
- 채움채(두부류) 발주를 고정 5품목 체크리스트로 + 외부 시스템 자동발주(20:05)
- 예약 알림을 GitHub Actions 스케줄러로 전환
- 점주 알림 + 예약 알림 + 주말 운영시간 + 가맹점 1회발주 잠금
- 업자 웹푸시 알림(새 발주 도착)
- PWA 매니페스트 + 앱 아이콘(홈 화면 설치)

**발주 핵심 기능**
- 채팅(카톡)식 발주 입력 모드
- 회원 관리 + 관리자 발주목록 날짜선택 + 채팅 확인단계 인라인 편집
- 관리자 전체발주(업자 범위) vs 핫딜마켓 발주관리(카테고리 범위) 분리
- 핫딜마켓 발주 개편: KST 시간표시, 4종 동시발주 + 발주확인, 오후 8시 마감 + 카운트다운, 지난발주 날짜묶음
- 오더야 리브랜딩 + 운영시간/수정/합본 등 대규모 개편

**인프라**
- 커스텀 도메인 적용(www.orderya.xyz, 가비아 DNS)
- 하단 네비게이션 디자인 개선
- 새롭 관리자: 공구 발주를 메인으로, 관리는 하단 네비
